## Assumptions

- The harness's Node 18+ Next.js runtime exposes `fetch` with streaming + `AbortController`; no `axios` is required.
- Cysic's `/v1/chat/completions` accepts an OpenAI-style request body (`{model, messages, temperature?, max_tokens?, response_format?}`) and returns an OpenAI-style `choices[0].message.content` string. The plan is correct as long as that contract holds; if Cysic requires extra headers (e.g. `X-API-Key` instead of `Authorization: Bearer`), the header map is a single-line fix and is called out in Implementation Notes.
- The LLM is asked to return **strict JSON** that we can `JSON.parse` safely (we'll instruct it in the system prompt and validate with `zod`).
- Quality is evaluated server-side at request time. No long-lived LLM client/pool, no streaming back to the browser in v1.
- The new feature is **opt-in by env-var presence**: when `CYSIC_API_KEY` is unset, the tool returns a clear `503 llm_not_configured` error rather than crashing the server.
- The Cysic endpoint is reachable from the Vercel serverless runtime; the user is responsible for setting the env var in Vercel.

### Key technical decisions

- **Native `fetch` over `axios`**: zero added deps; Next 14's Node.js runtime already exposes global `fetch` and `AbortController`. The user's reference snippet's `npm install axios` step is dropped in favor of the built-in.
- **No streaming**: simplest v1. The route returns the full `QualityReport` once the LLM call completes. A 30 s `CYSIC_TIMEOUT_MS` cap is the right order of magnitude for a 2 048-token response.
- **Strict JSON output**: instructed in the system prompt ("Return a single JSON object, no markdown fences, no commentary") and re-validated with `zod`. If validation fails we throw `LlmSchemaError` and the route returns `500` so operators see a clean failure rather than silently dropping fields.
- **Schema-validated prompt**: the system prompt includes the exact `QualityReport` JSON shape with field types and allowed enums, dramatically improving first-shot parse success.
- **Defensive error handling**: three bespoke error classes let the REST route map cleanly to HTTP codes and the MCP tool map cleanly to `isError: true` JSON, without leaking internal stack traces.
- **Env-var presence as the kill switch**: missing `CYSIC_API_KEY` returns `503 llm_not_configured` instead of crashing the server or faking a response. CI builds and local `npm run build` keep working without the key.
- **Idempotent, additive changes**: the four existing MCP tools, the seven detectors, the `analyze` REST route, the demo "Analyze" flow, and all smoke/validate scripts continue to pass without modification.

### Risk controls

- **Cysic API drift**: if the response shape differs from `{ choices: [{ message: { content: string } }] }`, the `LlmSchemaError` will fire on the first real call. Implementation Notes flags this as a single-line fix in `parseAndValidate`.
- **Bearer vs. `X-API-Key`**: the user's snippet shows `Authorization: Bearer â€¦`. If Cysic instead expects `X-API-Key`, change the header map in `lib/llm-client.ts` â€” call this out in `claude.md`'s "LLM integration" subsection.
- **Token explosion**: the user's source can be tens of kilobytes. The system prompt includes a hard cap ("If the source exceeds 400 lines, summarize the first 60 / last 40 lines and reference line ranges for findings") and we also set `CYSIC_MAX_TOKENS=2048` on the request.
- **Secret leakage**: the key is never logged; error messages only contain the first 6 chars + `â€¦`. `env.example` ships with an empty value so it can be committed safely.
- **Vercel cold-start latency**: a single 2 048-token call to a public endpoint is well under Vercel's 10 s function timeout when the timeout is set to 30 s; we keep `runtime = "nodejs"` for the fetch streaming.
